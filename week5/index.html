    Errors and Handling Errors

    The properties that are generally safe to use are:

    • name property: returns the name of the error constructor function used as a string. Ex: 'Error' or 'ReferenceError'
    • message property: returns a description of the error and should be provided as an argument to the error constructor
    • stack property: returns a stack trace for that error. Not safe to use in production sites.
    • throw statementL lallows for any problmes in the code to be highlighted and dealt with
